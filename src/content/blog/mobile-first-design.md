      # Mobile-First Design: Why 85% of Nigerians Browse on Phone

      By Hephzibah Media Team · Published December 25, 2024
      85% of Nigerian internet users access the web exclusively through mobile. Yet most Nigerian websites look like they were designed for desktop first, then shrunk. This is backwards. Here's how to design for mobile from the ground up.
    
  

  
    
      ## The Mobile-First Philosophy

      
Build the mobile experience first  with constraints. Then expand to tablet and desktop. This forces you to prioritise ruthlessly: What's essential? What's a nice-to-have? This discipline creates better experiences for everyone. Example: A desktop-first site might have 10 navigation links; mobile-first sites typically have 56 essential links + a menu.

      ## Touch Targets: Make Buttons Big

      
On desktop, a 32px button is fine. On mobile (held in one hand), it needs 4856px minimum. Make links tappable, not clickable. Leave space between interactive elements  accidental taps are the #1 mobile frustration. Test on actual devices, not just browser emulation.

      ## Readable Text Without Zooming

      
Default font size: 16px minimum for body text. Don't force users to pinch-zoom to read. Line length: 3040 characters for mobile (shorter than desktop). Example: 'Lorem ipsum dolor sit amet, consectetur adipiscing elit.' reads differently on a 3-inch phone vs 24-inch monitor.

      ## Images: Responsive & Optimized

      
Use CSS media queries: 800px image on desktop, 300px on mobile. Use WebP format for mobile (40% smaller). Aspect ratios: 3:2 or 16:9 for news/blogs. 1:1 for product photos (easier to view on mobile). Use lazy loading for images below the fold  they load only when visible.

      ## Forms: Mobile-Optimized Input

      
Use appropriate input types:  opens numeric keypad,  shows @ symbol. Avoid password-masked fields for 'confirm password' on first submit. Progress indicators: let users know 'Step 2 of 3.' One question per screen beats long forms on mobile.

      ## Navigation: Hamburger vs Tabs

      
Hamburger menus hide navigation (saves space), but they're harder to discover. Tabs at the bottom (like Instagram, TikTok) are more discoverable but use space. Best practice: 56 essential navigation items visible, everything else in a menu. Test with real users.

      ## Performance: The #1 Mobile Constraint

      
Mobile users are often on 3G with latency 100200ms higher than desktop. Every KB matters. Lazy load images, defer JS, compress SVGs, minify CSS. Test on a real 3G connection, not just WiFi. This is where most Nigerian websites fail.
